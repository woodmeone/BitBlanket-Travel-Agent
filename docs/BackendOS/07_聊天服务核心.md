## 3. 核心流程总结

### 3.1 stream_chat 完整流程

```
stream_chat(message, session_id, mode, ...)
    │
    ├─ 1. bind_request_context()          ← 绑定请求追踪上下文
    │
    ├─ 2. _normalize_mode(mode)           ← 规范化模式，无效值回退 "react"
    │
    ├─ 3. _StreamRunState(session_id)     ← 创建可变状态累积器
    │
    ├─ 4. initialize()                    ← 延迟初始化（双重检查锁定）
    │
    ├─ 5. _ensure_session(session_id)     ← 解析/创建会话
    │
    ├─ 6. yield SSE: session_id 事件      ← 通知前端绑定会话
    │
    ├─ 7. _save_user_message()            ← 持久化用户消息
    │
    ├─ 8. _write_memory_user()            ← 写入记忆管理器
    │
    ├─ 9. _normalize_stream_events()      ← 【核心】按模式分支流式输出
    │      │
    │      ├─ direct: _normalize_direct_stream_events()
    │      │    ├─ yield reasoning_start/end
    │      │    ├─ yield answer_start
    │      │    └─ yield chunk* (LLM token 流)
    │      │
    │      ├─ react:
    │      │    ├─ yield reasoning_start
    │      │    ├─ _stream_agent_events() → _normalize_runtime_event()
    │      │    │    ├─ reasoning_chunk* (推理片段)
    │      │    │    ├─ subagent_start/end (子Agent生命周期)
    │      │    │    ├─ artifact_patch* (产物增量)
    │      │    │    ├─ tool_start/end (工具调用)
    │      │    │    └─ chunk* (回答 token)
    │      │    └─ _ensure_answer_section_started()
    │      │
    │      └─ plan:
    │           ├─ yield reasoning_start
    │           ├─ _normalize_plan_preview_events()  ← 先生成预览
    │           ├─ _stream_agent_events() → ...      ← 再执行 react 流程
    │           └─ _ensure_answer_section_started()
    │
    ├─ 10. _finalize_stream_run()         ← 持久化成功结果 + 构建 metadata + done
    │
    ├─ [异常] _finalize_stream_failure()  ← 持久化失败状态 + 构建 error + done
    │
    └─ finally: reset_request_context()   ← 清理请求上下文
```

### 3.2 _normalize_runtime_event 事件分发

```
Agent 运行时事件 (dict)
    │
    ▼
_normalize_runtime_event(state, event)
    │
    ├─ 1. _normalize_reasoning_runtime_event()
    │      ├─ type=="reasoning" → reasoning_chunk
    │      └─ type=="stage" → stage
    │
    ├─ 2. _normalize_subagent_runtime_event()
    │      ├─ type=="subagent_start" → subagent_start + 更新 state.subagent_events
    │      ├─ type=="subagent_end" → subagent_end + 更新 state.subagent_events
    │      └─ type=="artifact_patch" → artifact_patch + merge_artifact_payload()
    │
    ├─ 3. _normalize_tool_runtime_event()
    │      ├─ type=="tool_start" → tool_start + 追加 state.tools_used
    │      └─ type=="tool_end" → tool_end
    │
    ├─ 4. _normalize_answer_runtime_event()
    │      ├─ type=="chunk" → _ensure_answer_section_started() + chunk
    │      └─ type=="done" → _apply_done_event() (合并终态数据)
    │
    └─ 无匹配 → 返回空列表
```

### 3.3 三种聊天模式对比

```
┌─────────────┬──────────────────────────────────────────────────────┐
│   direct    │ 用户消息 → LLM 直接流式输出 → chunk* → done         │
│             │ 无工具调用，无推理过程，适合简单问答                    │
├─────────────┼──────────────────────────────────────────────────────┤
│   react     │ reasoning_start → Agent 推理+工具调用 → chunk* → done│
│             │ ReAct 循环：思考→行动→观察，支持工具和子 Agent         │
├─────────────┼──────────────────────────────────────────────────────┤
│   plan      │ reasoning_start → plan_preview → Agent 推理+工具     │
│             │ → chunk* → done                                     │
│             │ 先展示计划预览（意图+步骤），再逐步执行                  │
└─────────────┴──────────────────────────────────────────────────────┘
```

## 4. 关键设计点

| 设计点 | 实现 | 为什么 |
|--------|------|--------|
| Mixin 组合模式 | `ChatService(StreamMixin, HistoryMixin, HealthMixin)` | 将复杂类拆分为独立 Mixin，每个 Mixin 职责单一，可独立测试和演进，避免"上帝类" |
| 延迟初始化 | `initialize()` 使用双重检查锁定 | LLM 适配器和 Agent 运行时初始化较重（加载模型、注册工具），延迟到首次请求时才执行，加快启动速度 |
| _StreamRunState | dataclass 累积可变状态 | 每次请求创建独立实例，在流式过程中逐步填充，最终用于持久化和终端事件构建 |
| 事件处理器链 | `_normalize_runtime_event` 按优先级尝试4个处理器 | 避免巨型 if-elif 链，每个处理器职责清晰，新增事件类型只需添加新处理器 |
| _ensure_answer_section_started | 自动补充推理→回答边界事件 | Agent 可能直接产出回答而未推送 reasoning_end，自动补充保证前端状态机正确转换 |
| 记忆写入容错 | `_write_memory_user/assistant` 吞并异常 | 记忆写入失败不影响主聊天流程，仅记录警告日志 |
| SLO 滑动窗口 | `_health_metrics` deque + 时间窗口清理 | 只统计近期请求的健康指标，过期数据自动清理，确保指标反映真实近期状态 |
| 双重检查锁定 | `initialize()` 先快速检查标志，再加锁检查 | 避免已初始化后的锁开销，同时防止并发场景下重复初始化 |
| 工具结果截断 | `_stream_agent_events` 中截断 tool_end 结果 | 防止过长的工具输出占用 SSE 带宽，前端只需预览摘要 |
