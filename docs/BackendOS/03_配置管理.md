## 4. 关键设计点

| 设计点 | 实现方式 | 为什么 |
|--------|----------|--------|
| **环境变量替换先于 YAML 解析** | `_replace_env_vars()` 在 `yaml.safe_load()` 之前执行（L80→L82） | 替换发生在文本层面，确保 YAML 解析器看到的是实际值而非占位符。如果先解析再替换，`${VAR}` 可能被 YAML 解析器误处理 |
| **环境变量不存在时保留占位符** | `return env_value if env_value else match.group(0)`（L113） | 优雅降级：缺失环境变量不导致启动失败，而是保留占位符供后续 `_is_model_active()` 判断该模型不可用 |
| **`yaml.safe_load` 而非 `yaml.load`** | L83 使用 `safe_load` | `yaml.load` 可以执行任意 Python 代码（安全风险），`safe_load` 只解析基本数据类型 |
| **点分键路径查询** | `get_config("web.port")` → `config["web"]["port"]`（L117-139） | 避免调用方写多层 `config.get("web", {}).get("port")`，提供更简洁的访问方式 |
| **`_is_model_active()` 三重检查** | 空值→占位符→模板提示（L173-183） | 防止前端展示不可用的模型。空值=未配置，占位符=环境变量缺失，`YOUR_` = 用户未替换模板值 |
| **两种单例实现** | 模块级变量 vs `@lru_cache` | 模块级变量更灵活（可手动 reset），`@lru_cache` 更简洁。`runtime.py` 用 `@lru_cache` 因为不需要手动重置 |
| **懒加载避免循环依赖** | `get_server_config()` 在函数内部 `from config import server_config`（L52） | 如果在文件顶部 import，`config.runtime` 和 `config.server_config` 可能互相引用导致 `ImportError`。函数内 import 确保调用时两个模块都已加载完成 |

### 深入理解：懒加载如何解决循环依赖

```python
# 问题场景：
# config/runtime.py 顶部: from config import server_config
# config/server_config.py 顶部: from config.runtime import get_model_config_manager
#
# Python 加载 config.runtime 时：
#   1. 执行 from config import server_config
#   2. Python 开始加载 config.server_config
#   3. 执行 from config.runtime import get_model_config_manager
#   4. 但 config.runtime 还没加载完！→ ImportError
#
# 解决方案：懒加载
# config/runtime.py:
def get_server_config():           # L45
    from config import server_config  # L52: 函数内 import
    return server_config
#
# 现在加载 config.runtime 时不会触发 server_config 的加载
# 只有调用 get_server_config() 时才 import，此时两个模块都已加载完成
```

### 深入理解：环境变量替换的完整示例

```yaml
# llm_config.yaml 原始内容：
models:
  gpt-4o:
    api_key: ${OPENAI_API_KEY}
    provider: openai
  qwen-plus:
    api_key: ${DASHSCOPE_API_KEY}
    provider: dashscope
  demo-model:
    api_key: YOUR_API_KEY_HERE
    provider: openai

# 环境变量设置：
# OPENAI_API_KEY=sk-xxxxxxxx
# DASHSCOPE_API_KEY 未设置

# _replace_env_vars() 处理后：
models:
  gpt-4o:
    api_key: sk-xxxxxxxx           # ← 已替换
    provider: openai
  qwen-plus:
    api_key: ${DASHSCOPE_API_KEY}  # ← 保留占位符
    provider: dashscope
  demo-model:
    api_key: YOUR_API_KEY_HERE     # ← 不变
    provider: openai

# _is_model_active() 判断结果：
# gpt-4o:     ✅ 可用（api_key 是有效值）
# qwen-plus:  ❌ 不可用（api_key 仍是 ${VAR} 占位符，环境变量不存在）
# demo-model: ❌ 不可用（api_key 包含 "YOUR_" 模板提示）
```
