## 4. 关键设计点

| 设计点 | 实现方式 | 为什么 |
|--------|----------|--------|
| **uvicorn + FastAPI 分离** | uvicorn 是 ASGI 服务器，FastAPI 是 Web 框架，两者通过 ASGI 协议协作 | 职责分离：uvicorn 负责网络 I/O（接收请求、管理连接），FastAPI 负责业务逻辑（路由、校验、序列化）。类似 Nginx + Node.js 的关系 |
| **模块级 `app` 变量** | `app = create_app()` 在模块加载时执行（L34） | uvicorn 通过字符串 `"moyuan_web.main:app"` 定位 FastAPI 实例，要求 app 在模块导入时即可用。这是 uvicorn 的约定 |
| **`ensure_project_paths()`** | 将 `PROJECT_ROOT` 和 `BACKEND_ROOT` 插入 `sys.path` 开头（L33-35） | 项目结构深（`agent/travel_agent/...`），Python 默认的搜索路径找不到这些包。手动插入确保 `from agent.travel_agent...` 能正常工作 |
| **配置文件优先于 CLI 参数** | L51-54：仅在 CLI 使用默认值时才用配置文件覆盖 | 用户显式指定的参数应优先，配置文件作为"默认值的默认值"。避免配置文件覆盖用户意图 |
| **`PROJECT_ROOT` 的计算** | `Path(__file__).resolve().parents[2]`（L19） | 从 `backend/moyuan_web/bootstrap.py` 向上 2 层到达项目根目录。不硬编码路径，确保在不同机器上都能正确解析 |

### 深入理解：uvicorn 与 FastAPI 的关系

```
  HTTP 请求
     │
     ▼
┌──────────┐    ASGI 协议    ┌──────────┐
│  uvicorn │ ──────────────→ │  FastAPI  │
│ (服务器)  │ ←────────────── │ (框架)    │
└──────────┘   HTTP 响应     └──────────┘

uvicorn 做的事：              FastAPI 做的事：
- 监听端口                    - 路由匹配
- 接收 TCP 连接              - 请求校验 (Pydantic)
- 解析 HTTP 报文             - 调用业务函数
- 管理连接池                  - 序列化响应
- 热重载 (reload)            - 生成 OpenAPI 文档
```

### 深入理解：sys.path 与模块导入

```python
# Python 的 import 查找顺序：
# 1. sys.path[0]  → 当前脚本所在目录
# 2. sys.path[1:] → PYTHONPATH 环境变量 + 标准库 + site-packages
# 3. 找不到 → ModuleNotFoundError

# 本项目的问题：
# moyuan_web/main.py 中需要 from agent.travel_agent... 导入
# 但 agent 包在项目根目录下，不在 moyuan_web 的父目录中
# 所以需要 ensure_project_paths() 将项目根目录加入 sys.path

sys.path.insert(0, "/project/moyuan-travel-agent")        # 找到 agent 包
sys.path.insert(0, "/project/moyuan-travel-agent/backend") # 找到 moyuan_web 包
```
