# Product Requirements

## 产品定位

ShuaiTravelAgent 的目标不是做一个“会回答旅行问题的聊天框”，而是做一个更接近真实决策场景的 AI 旅行助手：帮助用户从模糊需求走到可执行方案。

## 目标用户

### 1. 直接想出发的普通用户

特点：

- 没有明确目的地
- 想快速得到一个靠谱方案
- 关注预算、轻松程度、雨天备选、适合同行人

### 2. 已有目的地但不想自己做攻略的人

特点：

- 希望减少查资料、做表格、比价格的时间
- 希望一份回答里直接包含预算、路线、注意事项、执行清单

### 3. 需要和同行人一起决策的人

特点：

- 需要并排对比几个城市或几套方案
- 需要方便分享给别人看

## 产品目标

### 目标 1：把长文本答案变成可继续操作的结果

不仅生成 Markdown，还要继续结构化成：

- 每日行程卡
- 预算卡
- 多方案对比
- 冲突检测
- Checklist
- 出发提醒
- 可导出分享的结果卡

### 目标 2：降低旅行决策成本

在用户真正出发前，帮助他们减少：

- 不确定去哪
- 不知道是否超预算
- 不知道是否适合亲子/老人/无车
- 不知道路线是否合理
- 不知道如何分享给同行人决策

### 目标 3：提高结果可信度

对于时效性强、风险高的问题：

- 让系统尽量做验证
- 显示 stale / fallback / risk 提示
- 让用户知道哪些地方需要二次确认

## 当前功能范围

### 对话体验

- `direct / react / plan` 三模式
- SSE 流式阶段与工具事件
- 多会话与模型切换
- 约束条件面板
- 首页示例问题入口

### 行程结果工具箱

- 每日行程卡
- 时间轴结构化展示
- 预算滑杆
- 可信度条与风险提示
- 多方案对比
- 冲突检测与一键修复
- 候选池与景点决策卡
- Checklist / 出发提醒
- 导出图片 / 分享短链

### 城市探索

- 100+ 城市探索池
- 地区 / 标签 / 快筛条件
- 城市卡片与推荐理由
- 收藏池 / 对比池
- 一键继续生成具体行程

### 地图与路线

- 真实路线距离预览
- 静态地图与路线 polyline
- 按距离重排

## 体验原则

### 1. 先降低门槛，再开放复杂能力

- 首页有 3 秒上手示例
- 用户可先问，再逐步补预算和约束
- 复杂工具放在结果区继续操作

### 2. 结果必须面向执行

- 不是只给推荐理由
- 要能继续导出、分享、执行、调整

### 3. 让风险可见，而不是假装确定

- 可信度、验证状态、fallback、时效风险要可视化

## 近期重点方向

- 继续增强地图与路线编辑体验
- 继续扩展城市探索内容质量与排序方式
- 优化分享页与对外查看体验
- 持续提高 Agent 在高风险问答场景下的验证稳定性
